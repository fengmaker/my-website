这是一个非常经典且痛点明确的问题。作为工业工程（IE）专业的学生，你可以把**网格 DP** 看作是一个**流水线作业（Pipeline）**。

你现在的痛点在于：**“记忆化搜索”逻辑很自然（自顶向下），但转成“递推数组”（自底向上）时，经常因为边界处理（`i-1 < 0`）、下标偏移（`dp[i]` 对应 `grid[?`）搞得心态爆炸。**

为了解决这个问题，我为你总结了一套**“加一行，加一列”的哨兵（Padding）通用模板**。这套方法能最大程度消除边界判断，统一递推公式。

---

### 一、 核心心法：下标偏移与物理意义

#### 1. 为什么会感到混乱？

在记忆化搜索中，你关注的是 dfs(i, j)，也就是“我现在在 (i,j)，我要从 (i-1, j) 拿数据”。

在普通递推中，如果不加 padding，你需要写大量的 if i == 0 或 if j == 0 来处理边界，导致代码逻辑分叉。

#### 2. 解决方案：Padding（哨兵法）

我们在 DP 数组的最上面加一行、最左边加一列（甚至右边），把它们作为“缓冲带”。

- **Grid 坐标系**：物理世界，范围 `0` 到 `m-1`。
    
- **DP 坐标系**：虚拟世界，范围 `0` 到 `m`（大小为 `m+1`）。
    
- **映射关系**：**`dp[i+1][j+1]` 对应 `grid[i][j]`**。
    

**口诀：**

> DP 下标多加 1，Grid 下标照旧取。
> 
> 当前状态看 i+1，前置状态看 i。

### 二、 通用模板归类

我们把你列出的题目分为三类，每类只要改一点点初始化逻辑。

#### 第一类：标准移动（右/下）

**题目：**

- 62. 不同路径（求方案数）
        
- 63. 不同路径 II（有障碍）
        
- 64. 最小路径和（求最值）
        
- 3393. 统计异或路径（带状态维度的方案数）
        

**标准化写法（以 64 题最小路径和为例）：**

1. **定义 DP 数组**：大小 `(m+1) x (n+1)`。
    
2. **初始化哨兵**：
    
    - 求**最小**值：哨兵填 `Infinity`（无穷大），这样取 `min` 时不会选到哨兵。
        
    - 求**方案数**：哨兵填 `0`。
        
3. **初始化起点**：
    
    - 手动设置 `dp[0][1]` 或 `dp[1][0]` 为特定值，或者直接在循环里特判 `dp[1][1]`。
        
    - _推荐技巧_：对于求最小值的题，把 `dp[0][1] = 0` (假设它是入口前的虚拟点) 往往能让逻辑统一，但最稳妥的是**循环内直接处理起点**。
        

Python

```
# 64. 最小路径和 (Minimum Path Sum)
def minPathSum(self, grid: List[List[int]]) -> int:
    m, n = len(grid), len(grid[0])
    # 1. 申请 m+1, n+1 的空间，默认值为“无效值”
    # 求最小值，无效值就是无穷大
    dp = [[float('inf')] * (n + 1) for _ in range(m + 1)]
    
    # 2. 循环遍历 Grid (i, j 从 0 到 m-1, n-1)
    for i in range(m):
        for j in range(n):
            # 3. 映射关系：dp[i+1][j+1] 对应 grid[i][j]
            
            # 特殊处理起点
            if i == 0 and j == 0:
                dp[i+1][j+1] = grid[0][0]
                continue
            
            # 4. 递推公式 (完全不需要判断边界！)
            # 只需要看 dp[i][j+1] (上方) 和 dp[i+1][j] (左方)
            # 因为我们有哨兵行和哨兵列，绝对不会越界
            dp[i+1][j+1] = min(dp[i][j+1], dp[i+1][j]) + grid[i][j]
            
    return dp[m][n]
```

**区别点：**

- **62. 不同路径**：dp 初始化为 `0`，`dp[1][1] = 1`，递推式是 `+` 而不是 `min`。
    
- **63. 不同路径 II**：如果 `grid[i][j] == 1` (障碍)，直接 `continue` (保持 dp 为 0)，否则正常递推。

#### 第二类：三角形/可变列宽

**题目：**

- 120. 三角形最小路径和
        

这题的特点是每一行长度不一样，且 (i, j) 来自 (i-1, j) 和 (i-1, j-1)。

痛点：j 可能等于 i（也就是上一行的末尾），此时 (i-1, j) 越界。

IE 优化思路：把三角形补成正方形（或者给每一行左右都加哨兵）。

最简便写法：直接在 dp 每一行最左和最右各加一个 inf。

Python

```
# 120. Triangle
# 既然是求最小，我们还是用 dp[m+1][m+1] 并填满 inf
dp = [[float('inf')] * (len(triangle) + 1) for _ in range(len(triangle) + 1)]
# 起点初始化
dp[1][1] = triangle[0][0] 

for i in range(1, len(triangle)): # 从第二行 grid 开始
    for j in range(len(triangle[i])):
        # dp index = grid index + 1
        # 你的左上方是 dp[i][j]，正上方是 dp[i][j+1] (注意这里是因为三角形斜着排的视觉误差，按坐标算是 j-1 和 j)
        # 修正：题目定义是 (i, j) 来自 (i-1, j) 和 (i-1, j-1)
        # 映射到 padded dp: dp[i+1][j+1] 来自 dp[i][j+1] 和 dp[i][j]
        dp[i+1][j+1] = min(dp[i][j+1], dp[i][j]) + triangle[i][j]
```

#### 第三类：复杂移动（下降路径）

**题目：**

- 931. 下降路径最小和（可以走左下、下、右下）
        

痛点：grid[i][j] 依赖 grid[i-1][j-1], grid[i-1][j], grid[i-1][j+1]。左右边界都会越界。

IE 优化思路：左右两边都加 Padding（各加一列 inf）。

Python

```
# 931. Minimum Falling Path Sum
# 宽度 n，我们在左右各加一列哨兵，宽度变成 n+2
dp = [[0] * (n + 2) for _ in range(m)] # 也可以不用行padding，只用滚动数组

# 初始化第一行 (注意下标偏移，dp的第 j+1 列对应 grid 的第 j 列)
for j in range(n):
    dp[0][j+1] = grid[0][j]
# 设置左右墙壁为无穷大
for i in range(m):
    dp[i][0] = float('inf')
    dp[i][n+1] = float('inf')

for i in range(1, m):
    for j in range(n):
        # dp[i][j+1] 对应当前 grid[i][j]
        # 上一行是 i-1
        # 三个来源：左上(j), 正上(j+1), 右上(j+2)
        dp[i][j+1] = min(dp[i-1][j], dp[i-1][j+1], dp[i-1][j+2]) + grid[i][j]

return min(dp[m-1]) # 取最后一行最小
```

### 三、 终极速查表 (Cheatsheet)

在做网格 DP 时，直接在纸上画出这个映射，就不会乱了：

1. **空间申请**：`dp = [[INIT] * (n + 1) for _ in range(m + 1)]`
    
    - 求最小：`INIT = inf`
        
    - 求数量：`INIT = 0`
        
2. **循环范围**：`for i in 0..m-1`, `for j in 0..n-1`
    
3. **核心公式**：
    
    - 左边项：`dp[i+1][j+1]`
        
    - 右边项（依赖项）：`dp[i][j+1]` (上), `dp[i+1][j]` (左)
        
    - Grid 值：`grid[i][j]` (完全没变，直接取)
        
4. **起点处理**：循环内 `if i==0 and j==0: dp[1][1] = ...; continue`
    

IE 学生的直觉理解：

把 Grid 想象成一批待加工的零件。

把 DP Table 想象成这就放置零件的托盘（Pallet）。

托盘比零件大一圈（Padding），是为了防止零件在加工（计算）过程中掉出去（越界）。你总是从托盘的 (1,1) 位置开始放第 (0,0) 号零件。

作为工业工程背景的学生，你可以把 **网格图 DP** 理解为 **在二维离散空间上的多阶段决策过程**。

以下是为你总结的网格图 DP 算法笔记，重点解决 **记忆化搜索（DFS）的设计**、**方向的选择** 以及 **如何转递推**。

### 一、 核心心法：网格 DP 的两种视角

网格 DP 的本质是在网格上走格子，每一步都在做选择（向左、向下、选哪个数等）。

#### 1. 记忆化搜索 (Top-Down, DFS)

- **心态**：“我现在站在 $(i, j)$，**未来**怎么走最好？”
    
- **定义**：`dfs(i, j)` 通常表示从 $(i, j)$ 出发到达终点的最优解。
    
- **优势**：逻辑符合人类直觉，不需要关心计算顺序（由递归栈自动处理），处理“稀疏图”或“非法状态多”的情况时效率高。
    

#### 2. 递推 (Bottom-Up, Iteration)

- **心态**：“我现在站在 $(i, j)$，我是从**过去**哪里走过来的？”
    
- **定义**：`dp[i][j]` 通常表示从起点走到 $(i, j)$ 的最优解。
    
- **优势**：没有递归开销，容易进行空间优化（滚动数组），是工程实现的标准写法。
    

---

### 二、 记忆化搜索 (DFS) 设计指南

当你拿到一道新题，先不要急着写 `for` 循环，先设计 DFS 函数。

#### 1. 参数设置：我们需要什么信息？

DFS 的参数决定了 DP 状态的维度。

通用公式：dfs(i, j, [constraints])

- **基础参数 `(i, j)`**：
    
    - **含义**：当前机器人在网格的第 $i$ 行，第 $j$ 列。
        
    - **适用**：62, 63, 64, 120, 931 等基础题。
        
- **附加约束参数 `k`**：
    
    - **含义**：除了位置，还有什么资源限制？（如：剩余步数、剩余消除障碍次数、当前金币数）。
        
    - **适用**：
        
        - **3418. 机器人获得的最大金币**：可能需要记录“剩余免疫感电次数”。
            
        - **2684. 移动最大次数**：可能需要记录“上一步的值”来判断当前是否递增（或者直接隐式包含在逻辑里）。
            
- **路径状态参数 `mask` / `current_val`**：
    
    - **含义**：路径的累积属性（如：路径异或和、乘积状态）。
        
    - **适用**：
        
        - **3393. 统计异或值**：需要 `dfs(i, j, current_xor)`。此时 `current_xor` 也是状态的一部分。
            
        - **1594. 最大非负积**：需要返回 `min` 和 `max`，或者参数带一个状态标记。
            

#### 2. 返回值定义：我们要解决什么？

- **求最值**：返回从当前点到终点的最大/小路径和。
    
- **求方案数**：返回从当前点到终点的路径数量。
    
- **求可行性**：返回 True/False。
    

#### 3. 递归过程 (Transition)

思考模板：

Python

```
# 伪代码
def dfs(i, j, k):
    # 1. 越界/Base Case 处理
    if 越界 or 障碍: return 无效值(INF or 0)
    if 到达终点: return 终点值
    
    # 2. 查表 (记忆化)
    if memo[i][j][k] != -1: return memo[i][j][k]
    
    # 3. 做决策 (核心逻辑)
    # 比如：只能向下或向右
    res = max(
        dfs(i+1, j, k) + grid[i][j],  # 向下走
        dfs(i, j+1, k) + grid[i][j]   # 向右走
    )
    
    # 4. 存表并返回
    memo[i][j][k] = res
    return res
```

### 三、 从 DFS 到 递推 (DP Table) 的转化方法

一旦你的 DFS 跑通了，转递推就是 **机械劳动**。

#### 1. 维度映射

- DFS 参数有几个变量，DP 数组就有几维。
    
- `dfs(i, j)` -> `dp[i][j]`
    
- `dfs(i, j, k)` -> `dp[i][j][k]`
    
- `dfs(i, j, xor_val)` -> `dp[i][j][xor_val]` (注意 xor_val 的范围)
    

#### 2. 循环方向 (核心难点：前/后)

这里的方向指的是 **填表的顺序**。

- **原则**：当你计算 `dp[i][j]` 时，它依赖的状态（比如 `dp[i-1][j]` 或 `dp[i+1][j]`）必须已经计算好。
    
- **常用模式 A（起点到终点，最常用）**：
    
    - 状态定义：`dp[i][j]` = 从起点到 $(i, j)$ 的最优解。
        
    - 依赖关系：$(i, j)$ 来自 $(i-1, j)$ 或 $(i, j-1)$。
        
    - 循环顺序：`i` 从小到大，`j` 从小到大。
        
- **常用模式 B（倒序，处理特殊依赖）**：
    
    - 有些题目（如 120. 三角形最小路径和）如果定义 `dp[i][j]` 为“从 $(i, j)$ 到底部的最小路径”，那么计算 `dp[i]` 时需要 `dp[i+1]` 的数据。
        
    - 循环顺序：`i` 从大到小。
        

#### 3. 边界处理 (Padding 技巧)

正如之前讨论的：

- **求和/求最值**：推荐使用 **Padding (哨兵)**。多开一行一列，填上 `INF` 或 `0`，避免大量的 `if i-1 < 0` 判断。
    
- **求乘积**：推荐 **手动初始化** 第一行第一列。

### 四、 针对你列表中的题目分类总结

根据你的题目列表，我把它们分为四类，每一类的解题模板略有不同：

    - **2684. 移动最大次数**：只能往值更大的格子走（这是一个隐式 DAG 图）。
        
- **特点**：通常是一行一行处理，每行依懒于上一行。
    
- **状态**：`dp[i][j]` 表示到达 $(i, j)$ 的最优解。
    
- **转化**：`dp[i][j] = grid[i][j] + min(dp[i-1][k] for k in valid_range)`
    

#### 3. 带额外维度的状态 (Constraints)

- **题目**：
    
    - **3418. 机器人最大金币** (假设带特殊技能)：如果题目允许“消除k次障碍”或“双倍金币k次”。
        
    - **2304. 网格最小路径代价**：代价不仅取决于格子，还取决于“从哪条边过来的”（类似图论）。
        
- **特点**：二维不够用，必须加第三维。
    
- **状态**：`dp[i][j][k]`。
    
- **思考**：`k` 是怎么消耗的？如果是消除障碍，`dp[i][j][k]` 来源于 `dp[prev][k]` (没用技能) 或 `dp[prev][k-1]` (用了技能)。
    

#### 4. 路径聚合属性 (Path Property)

- **题目**：
    
    - **1594. 最大非负积**：路径状态不仅是值，还有符号。需同时维护 `min` 和 `max`。
        
    - **3393. 统计异或值**：路径状态是 XOR 累积值。
        
- **特点**：值本身成为了索引。
    
- **状态**：
    
    - XOR 类：`dp[i][j][xor_val]` = 方案数。
        
    - 这意味着如果异或值范围是 0-15，你的 DP 数组就是 `[m][n][16]`。
        
- **转移**：`dp[i][j][curr ^ grid[i][j]] += dp[i-1][j][curr]`。

### 网格图 DP 进阶算法笔记

#### 一、 基础模型的变体（数值与状态扩展）

**核心思想**：不再是简单的 `dp[i][j] = dp[i-1][j] + dp[i][j-1]`，而是路径上的数值运算变得复杂，或者需要携带额外的“余数/状态”。

- **乘积最大/最小 (1594. 矩阵的最大非负积)**
    
    - **难点**：负负得正。
        
    - **技巧**：必须同时维护两个 DP 数组（或状态）：`max_val[i][j]` 和 `min_val[i][j]`。当前位置的最大值可能来自“前一个最大值 $\times$ 正数”，也可能来自“前一个最小值 $\times$ 负数”。
        
- **模运算状态 (2435. 矩阵中和能被 K 整除的路径)**
    
    - **状态**：`dfs(i, j, remainder)`。
        
    - **参数**：第三维 `remainder` 表示从起点走到当前，路径和模 K 的余数。
        
    - **转移**：`(curr_rem + grid[i][j]) % k` 传递给下一层。
        

#### 二、 隐式 DAG 与 记忆化搜索 (DFS + Memo)

代表作：329. 矩阵中的最长递增路径、2328. 网格图中递增路径的数目

核心思想：当依赖关系不是简单的“左边和上边”，而是由数值大小决定（比如滑雪、递增路径）时，网格图就变成了一个 DAG（有向无环图）。

- **写法选择**：**首选记忆化搜索**。因为迭代 DP 需要先对单元格数值排序（拓扑排序），写起来麻烦且容易错。
    
- **不需要 visited 数组**：因为严格递增的条件（`grid[next] > grid[curr]`）天然保证了不会走回头路，不会死循环。
    
- **返回值设计**：
    
    - 求长度 (329)：`return 1 + max(neighbors)`
        
    - 求数目 (2328)：`return 1 + sum(neighbors)` (1 代表路径只包含自己)
        

#### 三、 逆向思维与“未来对现在的影响”

代表作：174. 地下城游戏

核心思想：正向推导会有“后效性”问题（前面的路径分高，但可能导致后面死掉）。

- **方向**：从 **右下角 $\to$ 左上角** 推导。
    
- **状态定义**：`dp[i][j]` 表示 **从 (i, j) 走到终点，进入 (i, j) 瞬间所需要的最小生命值**。
    
- **方程**：`min_need = min(右边需求, 下边需求) - grid[i][j]`。
    
- **边界**：时刻保证 `dp[i][j] >= 1`（活着）。
    

#### 四、 括号与合法性剪枝

代表作：2267. 检查是否有合法括号字符串路径

核心思想：不仅要走通，还要维护路径的合法性。

- **状态**：`dfs(i, j, balance)`，`balance` 为左括号减右括号的数量。
    
- **剪枝 (Pruning)**：
    
    1. `balance < 0`：右括号多了，立刻剪枝（非法）。
        
    2. `balance > 剩余步数`：左括号太多，剩下的路全变成右括号也消不完，立刻剪枝。
        
- **注意**：倒着写（从终点往回）逻辑会反转，容易出错，建议**正向 DFS**。
    

#### 五、 降维打击：行列扫描优化

代表作：1937. 扣分后的最大得分

核心思想：当转移方程涉及绝对值 abs(j - k) 且需要枚举上一行所有 k 时，复杂度为 $O(M \cdot N^2)$，会超时。需要利用性质优化到 $O(M \cdot N)$。

- **拆解绝对值**：将 `max(dp[k] - |j - k|)` 拆解为：
    
    - $k \le j$ (左侧)：`max(dp[k] + k) - j`
        
    - $k > j$ (右侧)：`max(dp[k] - k) + j`
        
- **技巧**：
    
    - **左扫 (Left Pass)**：`L[j] = max(L[j-1] - 1, dp[i-1][j])`
        
    - **右扫 (Right Pass)**：`R[j] = max(R[j+1] - 1, dp[i-1][j])`
        
    - **合并**：`curr[j] = grid[i][j] + max(L[j], R[j])`
        

#### 六、 多智能体协同 (Multi-Agent / Cherry Pickup)

代表作：741. 摘樱桃、1463. 摘樱桃 II、3363. 最多可收集的水果数目

核心思想：将“去一趟再回一趟”转化为“两个人同时从起点出发”。

1. **参数设计（降维）**：
    
    - 不要写 `dfs(x1, y1, x2, y2)`（4维状态），容易爆内存。
        
    - 利用步数同步：`x1 + y1 = x2 + y2 = step`。
        
    - **优化状态**：`dfs(step, y1, y2)` 或 `dfs(x1, y1, y2)`。可以通过 `step` 和 `y` 算出 `x`。
        
2. **冲突处理**：
    
    - 如果 `y1 == y2`（两人走到同一格）：只加一次分 `grid[x][y]`。
        
    - 如果 `y1 != y2`：加两次分 `grid[x1][y1] + grid[x2][y2]`。
        
3. **区域限制与对称性 (3363 题特有)**：
    
    - 利用题目给的移动限制（只能左下、右下等），分析出每个人被锁死在特定区域（如上三角、下三角）。
        
    - **技巧**：利用矩阵转置 `zip(*grid)`，复用同一个 DFS 函数处理对称的区域。

### 通用代码模板与检查清单

#### 1. 递归函数的参数设计

- **一般情况**：`dfs(i, j)`
    
- **带状态**：`dfs(i, j, current_val)` (如余数、括号平衡、当前乘积)
    
- **双人/多人**：`dfs(step, col1, col2)` (用步数换算行坐标)
    

#### 2. 边界处理 (Base Cases)

- **越界检查**：放在最开头。
    
    Python
    
    ```
    if i < 0 or j < 0 or i >= m or j >= n: return -inf (求最大) / 0 (求和)
    ```
    
- **障碍物**：`if grid[i][j] == -1: return -inf`
    
- **剪枝**：`if balance > limit: return False`
    
- **终点**：
    
    - 如果求路径和/最大值：到达终点返回 `grid[i][j]`。
        
    - 如果求路径数：到达终点返回 `1`。
        
    - **易错点**：多人走到终点时，不要重复计算。
        

#### 3. 初始化与返回值

- **求最大值**：初始化 `ans = -inf`。
    
- **求最小值**：初始化 `ans = inf`。
    
- **求方案数**：初始化 `ans = 0`，注意取模 `MOD`。
    
- **记忆化**：Python 直接用 `@cache`，或者手写 `memo[i][j]`。**注意多组测试用例或多次调用时要 `cache_clear()`**。
    

#### 4. 你的专属易错点提醒

1. **参数漏加**：在 `return max(...)` 时，别忘了加上当前格子自己的值 `grid[i][j]`（除非是 174 这种逆向扣减的）。
    
2. **状态冲突**：在 1937 题中，不要把 `left` 和 `right` 简单相加，要取 `max`。
    
3. **多智能体撞车**：在 3363 题中，虽然几何上回不去，但加上 `if j <= i: return -inf` 显式避开重叠区域会让逻辑更严密。
