### 第一阶段：地基 —— 列生成 (Column Generation)

在碰 VRP 之前，必须先在一个简单问题上彻底搞懂列生成机制。**切割库存问题 (Cutting Stock Problem, CSP)** 是最好的磨刀石。

#### 1. 核心概念

- **Dantzig-Wolfe 分解**：如何把原问题拆成 RMP（限制主问题）和 Pricing（子问题）。
    
- **对偶原理**：为什么 $\pi_i$ 能告诉我们哪种切割方案最赚钱？
    

#### 2. 必读论文

- **Gilmore, P. C., & Gomory, R. E. (1961).** _"A linear programming approach to the cutting-stock problem"_.
    
    - **看点：** 历史上第一篇列生成论文。不用全读，看它是如何定义“一种切割模式”作为一列的。
        
- **教材章节：** _Desaulniers, G. et al. (2005). "Column Generation"_ -> **Chapter 1: The Primer**.
    
    - **评价：** 最好的入门教材，手把手推导。
        

#### 3. 通俗资源

- **视频（强烈推荐）：** YouTube 搜索 **"Column Generation widely explained"** (Gurobi 官方或大学课程)。
    
- **博客：** [Optimization 101: Column Generation](https://www.google.com/search?q=https://medium.com/%40Optimization101) (这类 Medium 文章通常有很好的图解)。
    

#### 4. 代码实践 (Python)

- **任务：** 不要用库，用 `gurobipy` 手写一个 CSP 求解器。
    
- **代码参考：** Gurobi 官方示例 `cutstock.py`。
    
    - 位置：安装目录下的 `examples/python/cutstock.py`。
        
    - **学习点：** 看它如何获取 `Constr.Pi`，以及如何 `Column.addTerms()`。

### 第二阶段：核心 —— 标签算法 (Labeling Algorithm)

这是 VRP 精确解的**心脏**。不懂这个，就永远只能在门外徘徊。

#### 1. 核心概念

- **ESPPRC**：带资源约束的最短路问题。
    
- **Dominance Rule (占优规则)**：如何删掉那些“既费钱又费时间”的路径标签，防止算法爆炸。
    

#### 2. 必读论文 (圣经)

- **Feillet, D. et al. (2004).** _"An exact algorithm for the elementary shortest path problem with resource constraints: Application to some vehicle routing problems"_.
    
    - **重点：** **Section 3**。仔细看它是如何定义 Label $L=(i, C, T, L)$ 的，以及 Dominance Rule 的数学不等式。这是所有现代 VRP 算法的祖师爷。
        

#### 3. 通俗资源

- **视频课程：** **Coursera - Discrete Optimization (Pascal Van Hentenryck)**  
    **视频课程：** **Coursera - 离散优化 (Pascal Van Hentenryck)**
    
    - 搜索章节：Routing -> Resource Constraints。教授会画图演示标签是如何在一个图上推进的。
        
- **图解：**
    
    - _(此处简述)_: 想象你在走迷宫，每到一个路口就记一个小本本（花费，载重）。如果你发现朋友也到了这个路口，且他的花费和载重都比你少，你就直接“自杀”（被 Prune），让你朋友继续走。
        

#### 4. 代码实践

- **库：** `cspy` (Constrained Shortest Path in Python)
    
- **链接：** [https://github.com/torressa/cspy](https://github.com/torressa/cspy)
    
- **任务：** 使用 `cspy` 的 `BiDirectional` 类解一个简单的带时间窗最短路。
    
- **学习点：** 它是如何实现 `Ref` (Resource Extension Function) 的。

### 第三阶段：SOTA 加速器 —— ng-route 松弛

为什么以前的算法只能算 50 个点，现在能算 300 个点？因为我们不再追求“完美无环”。

#### 1. 核心概念

- **Elementary (基本路径)**：绝对不走重复点（太难算，NP-Hard）。
    
- **None-Elementary**：允许随便走重复点（太好算，但 Bound 极差）。
    
- **ng-route**：折中方案。只记得最近走过的 $n$ 个点不许重复，其他的重复了也无所谓。
    

#### 2. 必读论文 (转折点)

- **Baldacci, R. et al. (2011).** _"New state-space relaxations for solving the vehicle routing problem"_.
    
    - **重点：** 只需要看关于 **ng-sets** 定义的那两页。不要被后面的数学公式吓倒，它的核心思想极其简单。
        

#### 3. 代码映射

- **Python 实现：** 在 `vrpy` 中，虽然它主要用 `cspy`，但理解 ng-route 的逻辑有助于你去配置高级求解器的参数。目前 Python 原生实现 ng-route 的较少，通常是在 C++ 层（如 Bapcod）实现的。

### 第四阶段：终极形态 —— Branch-Cut-and-Price (BCP)

这是目前学术界和工业界的最强形态。

#### 1. 核心概念

- **Subset Row Cuts (SR-Cuts)**：一种特殊的割平面，专门切那类“有些客户被服务了 0.5 次”的小数解。
    
- **Bidirectional Search**：从起点和终点同时开始搜标签，在中间汇合。
    

#### 2. 必读论文 (当前 SOTA)

- **Pessoa, A., et al. (2020).** _"A Generic Exact Solver for Vehicle Routing and Related Problems"_.
    
    - **地位：** 这篇论文描述了求解器 **VRPSolver (Bapcod)** 的原理。它是目前的王者。
        
    - **重点：** 浏览 Introduction 和 Conclusion，了解它是如何把各种变种问题（VRP, VRPTW, PDVRP）统一成模型架构的。
        

#### 3. 代码工具 (SOTA 神器)

- **项目：** `VRPSolverEasy`
    
- **GitHub:** [https://github.com/VRPSolver/VRPSolverEasy](https://www.google.com/search?q=https://github.com/VRPSolver/VRPSolverEasy)
    
- **特点：** 这是一个 Python 包，底层调用 Pessoa 团队写的 C++ Bapcod 求解器。
    
- **学习方式：**
    
    1. `pip install vrpsolvereasy`
        
    2. 运行它的 `CVRP.py` demo。
        
    3. **感受：** 你会发现同样是 Python，这个比 `vrpy` 快几十倍，因为它底层用了 ng-route 和双向标签。

### 总结：你的学习清单 (Checklist)

按这个顺序下载和学习，不要跳步：

1. **[Week 1 - 理论]** 阅读 Desaulniers 书中第一章，搞懂列生成原理。

5. **[Week 4 - 最终实战]** 使用 `VRPSolverEasy`。参考它的文档，尝试把你自己的毕设题目（盲盒销售相关的物流问题？）转化成它的模型格式。
