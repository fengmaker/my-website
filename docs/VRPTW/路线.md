### 第一阶段：精通精确算法（Current Focus - 2个月）

**目标：** 彻底搞懂列生成与分支定价，完成纯 Python 手写的 VRPTW 求解器。这是你面试时的“核武器”。

- **理论补强：**
    
    - **线性规划对偶理论：** 能够不假思索地写出标准型 LP 的对偶问题（参考刚才的“三步走”）。
        
    - **Dantzig-Wolfe 分解：** 理解为什么列生成有效，主问题（RMP）和子问题（Pricing）的数学界限。
        
    - **最短路算法进阶：** 深入研究 **ESPPRC** (Elementary Shortest Path Problem with Resource Constraints)。这不是普通的 Dijkstra，而是基于 **Label Setting** 的算法。
        
- **代码实战 (VRPTW Solver)：**
    
    - **Step 1:** 用 Gurobi/COIN-OR 调用 API 写一个 Compact Model（直接建模），作为 benchmark。
        
    - **Step 2 (核心):** 实现列生成框架。
        
        - 主问题：用求解器解松弛后的 RMP。
            
        - 子问题：**手写** Label Setting 算法求解 ESPPRC。
            
        - _难点攻克：_ 实现 **Dominance Rules (优势规则)** 来剪枝，否则节点稍微多一点就会跑不动。这是面试最爱问的细节。
            
    - **Step 3:** 加入分支定界 (Branching)。
        
        - 由于列生成涉及动态加列，不能直接对变量 $x_{ij}$ 分支，要学习 **Branch on Arc** 或 **Branch on Resource** 策略。
            
- **参考资源：**
    
    - 书籍：_Column Generation_ (Desaulniers et al.) —— 挑选 VRP 相关章节看。
        
    - 代码参考：GitHub 上搜 `pyvrp` (学习其结构) 或 `vrpy`。
        

### 第二阶段：掌握工业级启发式（Month 3 - 4）

**目标：** 解决大规模问题（1000+节点），这是企业实际业务的常态。

- **理论补强：**
    
    - **ALNS (自适应大邻域搜索):** 弄懂 Destroy 和 Repair 算子的设计逻辑，以及 Roulette Wheel Selection（轮盘赌选择）如何自适应调整算子权重。
        
    - **禁忌搜索 (Tabu Search):** 理解禁忌表 (Tabu List) 和藐视准则 (Aspiration Criteria)。
        
- **代码实战：**
    
    - **任务：** 在你的 VRPTW 项目中增加一个 `HeuristicSolver` 模块。
        
    - **实现：** 手写一个 ALNS 框架。
        
        - 设计 3-4 个 Destroy 算子（如：随机移除、最差移除、相关性移除）。
            
        - 设计 2-3 个 Repair 算子（如：贪心插入、Regret 插入）。
            
    - **对比实验：** 用你的 ALNS 跑大规模算例，用你的精确算法跑小规模算例，对比 Gap 和时间。
        
- **参考资源：**
    
    - 论文：Ropke and Pisinger (2006) 关于 ALNS 求解 VRPTW 的经典论文。
        

### 第三阶段：AI + OR 前沿探索（Month 5 - 6）

**目标：** 差异化竞争，复现学长简历中的“机器学习辅助”亮点。

- **理论补强：**
    
    - **机器学习基础：** 你的背景里已有相关课程，重点复习 LightGBM/XGBoost 和 GNN (图神经网络) 在组合优化中的应用。
        
    - **Learn to Branch/Prune:** 阅读相关综述，理解 ML 是如何指导搜索方向的。
        
- **代码实战 (二选一)：**
    
    - **方案 A (学长同款 - 推荐):** **ML 辅助列生成**。
        
        - 在做 Label Setting 时，用一个轻量级 ML 模型（如 MLP 或 GBDT）预测某条边是否可能出现在最优解中。如果概率低，直接剪枝，不进行 Label 扩展。
            
        - 效果：牺牲一点点最优性，换取数倍的速度提升。
            
    - **方案 B (强化学习):** **RL 构造初始解**。
        
        - 用 Pointer Network 或简单的 DRL 模型训练一个 Agent，由它生成 VRPTW 的初始路径，再扔给 ALNS 进行优化。
            
- **产出：** 一篇小论文（哪怕不发，写成技术报告放在 GitHub Readme 里也非常加分）。
