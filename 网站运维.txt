这是一个为你定制的 **MkDocs 网站维护常用命令清单**。

建议你将这些内容复制，保存在你的 Obsidian 里的一个专门笔记中（比如叫 `网站运维.md`），或者贴在电脑前的便利贴上。

### 📌 核心命令清单 (CMD / 终端)

请确保在终端中已经进入了你的项目文件夹（即 `mkdocs.yml` 所在的目录）：
`cd D:\website\my-website`

#### 1. 日常写作与预览 (最常用)

这是你写笔记时的标准动作，用于开启本地服务器，实时查看网页效果。

```bash
mkdocs serve

```

* **作用：** 启动本地网站。
* **访问：** 浏览器打开 `http://127.0.0.1:8000`
* **退出：** 在终端界面按 `Ctrl + C`。

#### 2. 发布上线 (一键部署)

当你写完并检查无误后，用这个命令把网站推送到 GitHub Pages 给别人看。

```bash
mkdocs gh-deploy

```

* **作用：** 自动编译静态文件并推送到 GitHub 的 `gh-pages` 分支。
* **注意：** 运行此命令前，最好先确保代码已经用 Git 提交了（见下文）。

#### 3. 生成静态文件 (调试用)

如果你想看看生成的纯 HTML 文件长什么样，或者不通过 GitHub 部署而是要把文件传给别人。

```bash
mkdocs build

```

* **作用：** 会在当前目录下生成一个 `site` 文件夹，里面全是 HTML/CSS/JS 文件。

### 🔧 环境维护 (换电脑/重装时用)

如果你换了电脑，或者过段时间 Python 环境乱了，需要重新配置。

#### 4. 导出依赖列表 (备份环境)

为了防止以后忘了装过哪些插件（比如 `mkdocs-material`, `mkdocs-roamlinks-plugin`），建议生成一个清单。

```bash
pip freeze > requirements.txt

```

* **作用：** 把当前安装的所有 Python 插件包名写入 `requirements.txt` 文件。

#### 5. 一键安装依赖 (恢复环境)

在另一台电脑上，只需要这一句命令就能把所有插件装好。

```bash
pip install -r requirements.txt

```

---

### ☁️ Git 同步 (数据备份)

虽然 `mkdocs gh-deploy` 更新了**网页**，但你的**源码（Obsidian 笔记）**也需要备份到 GitHub 仓库里，防止电脑硬盘损坏数据丢失。

建议养成习惯，每次 `gh-deploy` 前后，顺手执行这一套“保存三连”：

```bash
git add .
git commit -m "更新了CVRP笔记和随笔"
git push

```

---

### 💡 极简工作流总结

作为一个每天都要写代码和论文的学生，推荐你养成这样的肌肉记忆：

1. **写笔记：** 打开 Obsidian 狂写。
2. **看效果：** 终端输入 `mkdocs serve`，浏览器确认公式和排版没乱。
3. **传云端：** 终端输入 `mkdocs gh-deploy` (发网站) + `git push` (存源码)。
