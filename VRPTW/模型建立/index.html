    模型建立

    好的，我们要正式进入现代 VRPTW 求解算法的核心领域了：Dantzig-Wolfe 分解 (Dantzig-Wolfe Decomposition) 与 列生成 (Column Generation)。

    这是从“业余玩家”迈向“专业运筹算法工程师”的分水岭。我们不再把视角局限在“点 \(i\) 到点 \(j\)”的微观操作上，而是站在“一整条路径”的宏观视角来看问题。

    1. 核心思想：视角的转换¶

    • 原始视角（Compact Model）： 像拼积木一样，一块一块地拼凑边 \((i, j)\)，还要时刻提防拼出来的东西超重或超时。

    • 集合划分视角（Set Partitioning）： 假设我已经拥有了一个巨大的路径库（比如有一亿条合法的路径）。我的工作变成了简单的“点菜”：从这一亿条路径里挑出几条，正好覆盖所有客户，且总成本最低。

    2. 主问题 (Master Problem, MP)¶

    我们将 VRPTW 建模为一个 集合划分问题 (Set Partitioning Problem, SPP)。

    2.1 参数定义¶

    首先，定义我们已知的“全宇宙”：

    • \(N\): 客户集合 \(\{1, 2, ..., n\}\)。

    • \(\Omega\): 所有满足时间窗和容量约束的合法路径的集合（这是个天文数字，我们不可能显式枚举出来，记在心里即可）。

    • \(r \in \Omega\): 集合中的某一条具体路径（例如：仓库 -> 客户1 -> 客户5 -> 仓库）。

    • \(c_r\): 路径 \(r\) 的总行驶成本（距离）。

    • \(a_{ir}\): 一个 0/1 参数（常量）。

      • \(a_{ir} = 1\): 表示路径 \(r\) 经过了客户 \(i\)。

      • \(a_{ir} = 0\): 表示路径 \(r\) 没经过客户 \(i\)。

    2.2 决策变量¶

    • \(\lambda_r \in \{0, 1\}\): 二进制变量。

      • \(\lambda_r = 1\): 选择路径 \(r\) 作为最终方案的一部分。

      • \(\lambda_r = 0\): 不选择路径 \(r\)。

    2.3 数学模型¶

    \[\begin{aligned} \text{(MP) } \min \quad & \sum_{r \in \Omega} c_r \lambda_r \quad & \text{(1. 最小化总成本)} \\ \text{s.t.} \quad & \sum_{r \in \Omega} a_{ir} \lambda_r = 1, \quad \forall i \in N & \text{(2. 覆盖约束)} \\ & \sum_{r \in \Omega} \lambda_r \le K \quad & \text{(3. 车辆数约束 - 可选)} \\ & \lambda_r \in \{0, 1\} \quad & \text{(4. 整数约束)} \end{aligned}\]

    模型解读：

    • 公式 (2): 这是核心。对于每一个客户 \(i\)，所有被选中的路径里，必须恰好有一条经过它。这就是“划分 (Partitioning)”的含义（不重不漏）。

      • 注：在实际计算中，为了数值稳定性，有时会改成 \(\ge 1\) (Set Covering)，但在理论推导中通常用 \(= 1\)。

    • 公式 (3): 限制使用的车辆总数不超过 \(K\)（如果有这个限制的话）。

    3. 分解与定价 (Decomposition & Pricing)¶

    3.1 为什么要分解？¶

    主问题看起来非常简洁漂亮（全是线性约束），但有一个致命伤：\(\Omega\) 太大了。对于 100 个点的 VRPTW，合法路径数可能是 \(10^{100}\) 量级。你无法把所有 \(\lambda_r\) 变量都放进求解器。

    3.2 限制主问题 (Restricted Master Problem, RMP)¶

    我们采取策略：只考虑一小部分路径。

    假设我们只找到了极少数几条路径，构成了子集 \(\Omega' \subset \Omega\)。我们用 \(\Omega'\) 构建的模型叫 RMP。

    求解 RMP，我们会得到：

    • 当前的解 \(\lambda^*\)。

    • 对偶变量 (Dual Variables) \(\pi_i\)：对应公式 (2) 中的每一个客户约束。

      • \(\pi_i\) 的物理含义：服务客户 \(i\) 的“影子价格”或“边际成本”。如果 \(\pi_i\) 很大，说明客户 \(i\) 很难被服务，或者服务他的成本很高。

    3.3 既约成本 (Reduced Cost)¶

    我们要想知道：在还没被发现的浩瀚路径海 \(\Omega \setminus \Omega'\) 中，有没有哪条新路径 \(r_{new}\)，如果把它加进 RMP，能让总成本下降？

    根据线性规划理论，如果一个变量 \(\lambda_r\) 的 既约成本 (Reduced Cost, \(\bar{c}_r\)) 为负数，那么让 \(\lambda_r\) 进基（变为 1）就有可能降低目标函数值。

    路径 \(r\) 的既约成本计算公式：

    \[\bar{c}_r = c_r - \sum_{i \in N} a_{ir} \pi_i\]

    • \(c_r\): 路径的真实物理成本（距离）。

    • \(\sum a_{ir} \pi_i\): 路径“赚取”的对偶价值总和。

    • 直观理解： 如果一条路径的物理成本很低，但它经过的客户对偶价格很高（很值钱），那么 \(Cost - Value < 0\)，这是一条“暴利”路径，必须把它找出来加入 RMP！

    4. 子问题 (The Pricing Subproblem)¶

    我们的目标变成了：寻找一条 \(\bar{c}_r < 0\) 的合法路径。

    为了方便求解，我们需要把 \(\bar{c}_r\) 的公式拆解回边的层面。

    假设路径 \(r\) 由一系列边 \((i, j)\) 组成，即 \(x_{ij}=1\)。

    • 路径真实成本：\(c_r = \sum_{(i,j) \in r} c_{ij}\)

    • 路径对偶价值：\(\sum_{i \in N} a_{ir} \pi_i = \sum_{(i,j) \in r} \pi_i\) （注意：这里 \(\pi_i\) 归属于节点 \(i\)，当车辆离开 \(i\) 前往 \(j\) 时，我们认为它“收集”了 \(i\) 的对偶值）。

    于是，边 \((i, j)\) 的修正权值 (Modified Edge Cost) 为：

    \[\bar{c}_{ij} = c_{ij} - \pi_i\]

    (注：通常规定 \(\pi_{depot} = 0\))

    4.1 子问题建模¶

    子问题是一个 带资源约束的最短路问题 (ESPPRC - Elementary Shortest Path Problem with Resource Constraints)。

    目标： 在图上找一条从 Depot 出发回到 Depot 的路径，使得路径上所有边的 \(\bar{c}_{ij}\) 之和最小（且为负）。

    约束条件：

    1. 流平衡： 从 Depot 出发，最后回到 Depot。

    2. 容量约束： 路径上所有客户的需求量之和 \(\le Q\)。

    3. 时间窗约束：

      • 到达客户 \(j\) 的时间 \(T_j \ge T_i + s_i + t_{ij}\)。

      • \(a_i \le T_i \le b_i\) （必须在时间窗 \([a_i, b_i]\) 内服务）。

      • 如果在 \(a_i\) 之前到达，必须等待。

    4. 基本路径约束 (Elementary)： 路径不能包含重复的客户（这是最难的约束）。

    5. 总结：完整的列生成循环¶

    这个过程就像总经理（主问题）和调度员（子问题）的对话：

    1. 初始化： 总经理手里只有几条很烂的路径（比如每辆车只送一个客户），构建 RMP。

    2. 求解 RMP： * 总经理用线性规划解 RMP。

      • 得到对偶价格 \(\pi\)：“客户 A 现在很贵，谁能顺路把 A 带上，我重赏！”

    3. 传达对偶： 总经理把 \(\pi\) 告诉调度员。

    4. 求解子问题（Pricing）：

      • 调度员更新地图上的边权：\(\bar{c}_{ij} = c_{ij} - \pi_i\)。

      • 调度员在地图上搜索最“赚”的路径（Reduced Cost 最负）。

    5. 判断：

      • 情况 A： 调度员找到了一条 \(\bar{c}_r < 0\) 的新路径。\(\to\) 添加列，把这条路径扔进 RMP，回到第 2 步。

      • 情况 B： 调度员找遍了所有可能，发现最短路的 \(\bar{c}_r \ge 0\)。\(\to\) 停止。当前的 LP 解已经是全宇宙最优的 LP 解了。

    6. 关键参数含义速查表¶

    符号 归属 含义 备注
    \(\lambda_r\) 主问题 决策变量：选不选路径 \(r\) 0 或 1（松弛后为 \(0 \le \lambda \le 1\)）
    \(c_r\) 主问题 参数：路径 \(r\) 的实际长度 由子问题算出来传给主问题
    \(\pi_i\) 主问题 结果：客户 \(i\) 的对偶价格 只要 RMP 一解完，马上得到，用于更新子问题边权
    \(\bar{c}_{ij}\) 子问题 动态边权：\(c_{ij} - \pi_i\) 核心！ 子问题是在这个权重的图上跑最短路
    \(Q\) 子问题 资源：车辆最大容量 ESPPRC 算法必须时刻检查 accumulation \(\le Q\)
    \([a_i, b_i]\) 子问题 资源：客户 \(i\) 的时间窗 ESPPRC 剪枝的强力依据
    \(s_i\) 子问题 参数：客户 \(i\) 的服务时间 到达 \(j\) 的时间计算需包含 \(s_i\)

    讲到这里，整个数学框架建立起来了吗？如果清晰了，下一步我们就要深入子问题的内部，看看到底怎么用 Label Setting Algorithm 来解这个 ESPPRC。这是写代码最难的部分。

    好，我们要拆解 VRPTW 求解器中最核心、最复杂、也最精妙的引擎：ESPPRC 子问题求解器。

    如果说主问题是“拼图”，那么子问题就是“动态规划 (Dynamic Programming)”。

    在普通的图算法（如 Dijkstra）中，一个节点只需要记录一个状态（最短距离）。但在 VRPTW 中，这行不通。为什么？因为即使你现在的距离很短，但如果你时间用完了，或者车装满了，这条路也就废了。

    所以，我们需要标签算法 (Label Setting Algorithm)。

    1. 什么是“标签” (Label)？¶

    你可以把“标签”想象成车辆走到某个点时的一张“状态快照”。

    一个标签 \(L\) 通常包含以下 5 个核心属性：

    \[L = (v, \bar{c}, t, q, S)\]

    1. \(v\) (Current Node): 我现在在哪？（例如：在客户 5 处）

    2. \(\bar{c}\) (Reduced Cost): 这一路走来，累积的既约成本是多少？（这是我们要最小化的目标，一定要用 \(c_{ij} - \pi_i\) 算）

    3. \(t\) (Time): 现在几点了？（必须满足 \(t \in [a_v, b_v]\)）

    4. \(q\) (Load): 车上已经装了多少货？（必须 \(\le\) 车辆容量 \(Q\)）

    5. \(S\) (Visited Set): 我之前去过哪些点？（用于防止走回头路，保证 Elementary）

    形象比喻： 每一个标签就是一个“平行宇宙”里的司机。在点 \(i\)，可能有 100 个不同的标签，代表 100 种到达 \(i\) 的不同方式（有的快但贵，有的慢但便宜，有的装得多，有的装得少）。

    2. 核心操作：资源扩展 (Resource Extension Functions, REFs)¶

    这是算法的“一步”。假设我们手里有一个在点 \(i\) 的标签 \(L_i\)，我们要尝试走到邻居点 \(j\)。我们需要计算出新的标签 \(L_j\)。

    扩展公式 (Extension Logic)：¶

    1. 成本更新：

      \[\bar{c}_j = \bar{c}_i + (c_{ij} - \pi_i)\]

      (注意：\(\pi_i\) 是点 \(i\) 的对偶价格，我们在离开 \(i\) 前往 \(j\) 时把它减掉，意味着“收获”了这个价值)

    2. 时间更新：

      \[t_{arrival} = t_i + s_i + t_{ij}\]
      \[t_j = \max(t_{arrival}, a_j)\]

      • \(s_i\): 在 \(i\) 的服务时间。

      • \(t_{ij}\): \(i\) 到 \(j\) 的行驶时间。

      • 核心逻辑： 如果早到了 (\(t_{arrival} < a_j\))，就等到 \(a_j\)。

    3. 载重更新：

      \[q_j = q_i + d_j\]

    4. 访问集合更新：

      \[S_j = S_i \cup \{j\}\]

    可行性检查 (Feasibility Check)：¶

    生成 \(L_j\) 后，必须立刻检查是否“暴毙”。如果满足以下任一条件，直接丢弃这个标签：

    1. 超时： \(t_j > b_j\) （到了 \(j\) 已经关门了）。

    2. 超载： \(q_j > Q\) （车炸了）。

    3. 重复访问： \(j \in S_i\) （这是个回头路）。

    3. 算法的灵魂：优势准则 (Dominance Rule)¶

    如果不加控制，标签的数量会呈指数级爆炸。每走一步，标签数就乘以 \(N\)。对于 100 个点，没几步内存就爆了。

    优势准则是唯一能救命的机制。

    逻辑： 如果标签 A 在所有方面都比标签 B 强（或一样强），那标签 B 就是个废柴，永远不可能扩展出比 A 更好的后代。删掉 B！

    判定公式：¶

    假设在同一个节点 \(v\)，有两个标签 \(L_1\) 和 \(L_2\)。

    如果同时满足以下 4 个条件，则称 \(L_1\) 支配 (Dominates) \(L_2\)：

    1. 成本更低： \(\bar{c}_1 \le \bar{c}_2\)

    2. 时间更早： \(t_1 \le t_2\) （时间越早，未来能去的点越多，灵活性越大）

    3. 载重更少： \(q_1 \le q_2\) （空位越多，未来能装的越多）

    4. 访问过的点是子集： \(S_1 \subseteq S_2\) （\(L_1\) 去过的地方 \(L_2\) 全去过，而且 \(L_2\) 还多去了别的地方，意味着 \(L_2\) 的未来选择更少）

    结论： 如果 \(L_1\) 支配 \(L_2\)，直接把 \(L_2\) 从内存中抹去。

    为什么时间窗在 VRPTW 里是“神队友”？

    因为时间窗越窄，不仅容易触发“可行性检查”直接杀掉标签，而且容易触发“优势准则”（更容易出现 \(t_1 \le t_2\) 的情况）。宽时间窗反而最难算。

    4. 算法流程 (Label Setting Algorithm)¶

    1. 算法流程（标签设置算法）

    这就好比是一个“升级版”的 Dijkstra。

    数据结构： 一个队列（或堆） \(E\)，里面放着待处理的标签。

    1. 初始化： 在 Depot (点0) 创建初始标签 \(L_0 = (0, 0, 0, 0, \{0\})\)。放入 \(E\)。

    2. 循环： 当 \(E\) 不为空时：

      • 取出 \(E\) 中 \(\bar{c}\) 最小（或 \(t\) 最小）的标签 \(L_{curr}\)，假设它在点 \(i\)。

      • 扩展 (Extend)： 对所有邻居 \(j\)：

        • 如果不违规（时间、容量、回路），生成新标签 \(L_{new}\)。

        • 支配检查 (Dominance Check)：

          • 检查 \(L_{new}\) 是否被点 \(j\) 已有的任何标签支配？如果是，丢弃 \(L_{new}\)。

          • 检查 \(L_{new}\) 是否支配点 \(j\) 已有的标签？如果是，把那些旧标签删掉。

        • 如果 \(L_{new}\) 存活，放入 \(E\)。

    3. 结束： 当 \(E\) 空了，检查所有到达 Depot (终点) 的标签。选出 \(\bar{c} < -0.0001\) 的那些，送回主问题。

    5. 进阶难点：Elementary 属性的代价¶

    在上述步骤中，最难处理的是 \(S\) (Visited Set)。

    • 检查 \(S_1 \subseteq S_2\) 是非常耗时的（位运算）。

    • 当点数超过 20-30 个时，记录完整的 \(S\) 会导致标签很难被支配（因为很难满足子集关系）。

    现代 SOTA 的处理方式 (ng-Route)：

    为了让支配更容易发生，我们故意记不清楚 \(S\)。

    • 我们只记录“最近走过的 \(k\) 个点”或者“特定邻域内的点”。

    • 这虽然会导致生成的路径可能有小环（relaxed），但能让算法速度提升几百倍，同时下界质量损失极小。这就是 ng-Route Relaxation 的本质。

    第一部分：主问题 (Master Problem) 的建模¶

    主问题本质上是一个集合划分问题 (Set Partitioning Problem)。

    1. 符号定义¶

    • \(\Omega\)：所有可行路径的集合（未知的，巨大的）。

    • \(r\)：集合中的某一条路径。

    • \(c_r\)：路径 \(r\) 的实际物理成本（距离）。

    • \(a_{ir}\)：系数矩阵。如果路径 \(r\) 经过客户 \(i\)，则 \(a_{ir}=1\)，否则为 0。

    • \(\lambda_r\)：决策变量。如果选择路径 \(r\)，则 \(\lambda_r=1\)，否则为 0。

    2. 限制主问题 (RMP) 数学模型¶

    在这个阶段，我们只拥有一个小子集路径 \(\Omega' \subset \Omega\)。我们对 \(\lambda\) 进行线性松弛（\(0 \le \lambda \le 1\)）。

    \[\begin{aligned} \min \quad & \sum_{r \in \Omega'} c_r \lambda_r \\ \text{s.t.} \quad & \sum_{r \in \Omega'} a_{ir} \lambda_r = 1 \quad \forall i \in N \quad \textcolor{red}{\longrightarrow \text{产生对偶变量 } \pi_i} \\ & \sum_{r \in \Omega'} \lambda_r \le K \quad \textcolor{red}{\longrightarrow \text{产生对偶变量 } \pi_0 (\text{车辆数约束})} \\ & \lambda_r \ge 0 \end{aligned}\]

    关键点： 当我们用单纯形法（Simplex）求解这个线性规划后，每一个约束都会对应产生一个对偶值 (Dual Value)。

    • \(\pi_i\)：代表服务客户 \(i\) 的“影子收益”。

    • \(\pi_0\)：代表使用一辆车的“影子成本”（通常是负数或0，这里暂且忽略，聚焦于 \(\pi_i\)）。

    第二部分：检验数 (Reduced Cost) 的推导¶

    这是面试和写论文时最需要讲清楚的数学逻辑。

    1. 单纯形法的基本原理¶

    在线性规划中，要判断是否有一个新的变量（列）\(\lambda_{new}\) 应该加入基（Basis），我们要看它的检验数 (Reduced Cost) 是否为负。

    检验数的通用公式是：

    \[\bar{c}_{new} = c_{new} - \mathbf{\pi}^T \mathbf{A}_{new}\]

    • \(c_{new}\)：新变量在目标函数里的系数。

    • \(\mathbf{\pi}\)：当前的对偶向量。

    • \(\mathbf{A}_{new}\)：新变量在约束矩阵里的列向量。

    2. 代入 VRPTW 场景¶

    对于我们想寻找的一条新路径 \(r\)：

    • 它的目标系数 \(c_r = \sum_{(i,j) \in r} c_{ij}\) （路径上所有边的物理距离之和）。

    • 它的列向量 \(\mathbf{a}_r\)：如果路径经过 \(i\)，分量为 1。

    • 对偶项 \(\mathbf{\pi}^T \mathbf{a}_r = \sum_{i \in N} \pi_i \cdot a_{ir}\) （路径上所有点的对偶值之和）。

    所以，路径 \(r\) 的检验数公式为：

    \[\bar{c}_r = \left( \sum_{(i,j) \in r} c_{ij} \right) - \left( \sum_{i \in \text{path}} \pi_i \right)\]

    3. 将检验数“下放”到边 (Edge)¶

    为了能用图算法求解，我们需要把上面的公式变形，把 \(\pi_i\) 分摊到每一条边上。

    我们将 \(\pi_i\) 绑定在离开节点 \(i\) 的动作上：

    \[\bar{c}_r = \sum_{(i,j) \in r} \underbrace{(c_{ij} - \pi_i)}_{\text{新的边权 } \bar{c}_{ij}}\]

    (注：通常设 \(\pi_{depot} = 0\))

    结论：

    我们在子问题中寻找“检验数最负”的路径，等价于在边权为 \(\bar{c}_{ij} = c_{ij} - \pi_i\) 的图上寻找最短路。

    第三部分：体现在标签算法 (Label Setting) 中的哪里？¶

    1. 子问题的定义¶

    子问题是在一个新的加权图 \(G'=(V, E)\) 上求解 ESPPRC。

def extend(label_from, to_node):
    from_node = label_from.current_node

    # --- 核心：检验数的体现 ---
    # 物理距离
    dist = distance_matrix[from_node][to_node] 

    # 减去出发点的对偶值（收集利润）
    # 如果 to_node 是仓库，它通常没有对偶值，或者设为0
    dual_val = duals[from_node] 

    # 计算新的边权 (Reduced Cost of the edge)
    reduced_edge_cost = dist - dual_val

    # 更新标签的累积 Cost
    new_cost = label_from.accumulated_reduced_cost + reduced_edge_cost
    # -----------------------

    # 其它资源的更新（时间、载重）用的是物理值，不受对偶影响！
    new_time = max(label_from.time + service_time[from_node] + time_matrix[from_node][to_node], early_window[to_node])
    new_load = label_from.load + demand[to_node]

    return Label(to_node, new_cost, new_time, new_load, ...)

    3. 为什么这一点至关重要？¶

    • 支配规则 (Dominance Rule)： 我们在比较两个标签优劣时，比的是 \(\bar{c}\) (Reduced Cost)，而不是物理 Cost。

    • 目标判断： 算法结束时，我们要找的是 \(\bar{c} < 0\) 的路径。如果算出来的 \(\bar{c} > 0\)，即使它是物理上的最短路，对主问题也没用，必须丢弃。
